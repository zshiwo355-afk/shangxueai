# 后端接口设计

> 本文档列出当前实际启用的 HTTP 路由分组与代表性接口，便于前端对接、CR 与排查。完整字段以 `backend/app/*_api.py` 中的 Pydantic schema 为准；本文不再列出所有字段。

通用约定：

- 所有 `/api/*` 响应均带 `Cache-Control: no-store`（见 [backend/app/main.py](backend/app/main.py)）。
- 鉴权：`Authorization: Bearer <JWT>` 或 `?access_token=...`；`require_admin` / `require_super_admin` 装饰受控接口。
- 列表接口统一支持 `page` / `page_size` / `keyword`，错误以 `HTTPException(detail)` 返回中文文案。

## 2. 鉴权（[auth.py](backend/app/auth.py)）

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/auth/login` | 账号密码登录，返回 JWT + user 快照 |
| GET | `/api/auth/me` | 当前登录用户 |
| POST | `/api/auth/logout` | 仅返回 ok（JWT 无状态） |
| GET | `/api/auth/providers` | 返回 `password_enabled` / `wecom_enabled` / `wecom_auto_redirect_in_client` |
| GET | `/api/auth/wecom/start?redirect=...` | 跳转企业微信授权页 |
| GET | `/api/auth/wecom/callback?code&state` | 企微回调，签发 JWT 并 302 到 `WECOM_FRONTEND_CALLBACK_URL` |

## 3. 训练（[training_api.py](backend/app/training_api.py)）

挂载在 `/api/training`，登录用户访问。

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| POST | `/api/training/start` | 开始训练，返回 `session_id`、`visible_brief`、`first_customer_message`、`state` |
| POST | `/api/training/chat` | 单轮对话，返回 `customer_reply` 与 `state` |
| POST | `/api/training/finish` | 结束训练，返回复盘 JSON 并写入 `training_records` |
| POST | `/api/training/reset` | 重置 session |

请求/响应字段对齐 [backend/app/schemas.py](backend/app/schemas.py) 中的 `TrainingStartRequest/Response`、`TrainingChatRequest/Response`、`TrainingFinishRequest/Response`。

- `POST /bulk-delete` / `POST /bulk-status` 批量操作。

## 7. 试卷（[papers_api.py](backend/app/papers_api.py)）

挂载在 `/api/admin/papers`。

- `GET /` / `POST /` / `GET /{id}` / `PUT /{id}` / `DELETE /{id}` 试卷 CRUD。
- `POST /bulk-delete` / `POST /bulk-status` 批量。

## 8. 试卷派发与提交（[paper_assignments_api.py](backend/app/paper_assignments_api.py)）

管理端 `/api/admin/paper-assignments`：

- `GET /` 派发列表，支持试卷/用户/状态筛选。
- `GET /pending-review` 待复核提交。
- `GET /pending-submissions` 待 AI/管理员处理的提交。
- `POST /` 批量派发。
- `DELETE /{assignment_id}` / `POST /bulk-delete` 删除/批量删除。
- `POST /{assignment_id}/wecom-push` / `POST /bulk-wecom-push` 单个/批量企微推送。
- `GET /{assignment_id}/submissions` 派发下的提交记录。
- `GET /submissions/{submission_id}` 提交详情。
- `POST /submissions/{submission_id}/grade` 管理员打分/复核。

用户端 `/api`（同文件 `submit_router`）：

- `GET /my-assignments`、`GET /assignments/{id}`、`POST /assignments/{id}/start`、`GET /assignments/{id}/...`、`POST /assignments/{id}/submit`、`GET /submissions/{submission_id}`。

## 16. 错误约定

- 401：未登录 / token 失效。
- 403：登录但无该业务/角色权限。
- 404：资源不存在或已删除。
- 409：状态冲突（比如已派发用户重复派发、试卷状态不允许操作）。
- 422：参数校验失败（FastAPI 默认）。
- 500：未捕获异常 / LLM / MaxKB / 企微返回异常时由对应模块映射成中文 detail。

前端 `frontend/src/lib/http.js` 已统一把状态码映射到中文文案，可直接 `message.error(err.message)`。
