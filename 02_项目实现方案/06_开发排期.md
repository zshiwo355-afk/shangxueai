# 开发里程碑与后续路线

> 项目早期按"10 天交付第一版"排期。当前已经完成多版本迭代，本文档不再保留按天排期，而是记录已落地的里程碑与后续可继续推进的方向。

## 一、已完成里程碑

### M1 训练核心（已完成）

- MaxKB 知识库导入与 `RuleLoader` 缓存。
- `POST /api/training/start | chat | finish | reset`，会话落 `training_sessions`。
- 工程级硬校验：`MIN_ROUNDS`、成交阈值、`hidden_training_pack` 不下发。
- React 训练准备页 / 聊天页 / 复盘页。

### M2 用户与鉴权（已完成）

- 账号密码登录 + JWT（30 天）+ 全局 401 跳登录。
- 用户管理 Tab：CRUD、Excel 批量导入、批量删除、部门/岗位下拉项维护。
- 超级管理员引导账号、白名单、角色 / 在职状态字段。

### M3 题库与试卷（已完成）

- `question_bank` / `papers` / `paper_questions` / `paper_assignments` / `paper_submissions` / `paper_answers` 全量落地。
- 管理端：题库 / 试卷 / 派发 / 待复核 / 题目导入；批量删除与批量改状态。
- 用户端：我的试卷、答题、提交结果。
- AI 主观题评分 worker（`paper_ai_worker`）。
- 截止提醒 worker（`deadline_reminder_worker`）。

### M4 场景通关（已完成）

- 通关库与批量派发；通关复盘走训练管道复用。
- 管理端待复核与人工评分；通关历史与详情页。

### M5 魔学院（已完成）

- 视频 / 视频系列 / 读物 / 读物系列 / 音频打卡 / 题点测验。
- 派发目标：全员 / 部门 / 岗位 / 在职状态 / 指定员工 / 白名单。
- 自动动作（`magic_auto_actions`）按规则触发派发。
- 视频封面、OSS 分片上传、补卡、Office Online 预览等增强。

### M6 企业微信（已完成）

- OAuth 登录：`/api/auth/wecom/start | callback`，前端 `WecomCallbackPage` 收尾。
- 应用消息推送：试卷派发 / 截止 / 提交 / 通关派发 / 通关截止 / 魔学院读物 / 视频。
- 通讯录同步：第三方 HR 拉取，管理端预览 → 执行；冲突分类（`employee_sync.py`）。
- 推送监控：列表 / 详情 / 统计 / 事件类型 / 批量删除。

### M7 素材库（已完成）

- 项目级素材组织、视频 / PDF / Office 文件预览与下载。
- OSS 分片上传与跨域代理。

## 二、当前的后台常驻任务

由 [main.py](backend/app/main.py) 启动钩子拉起：

1. `auto_action_worker`：魔学院自动动作。
2. `paper_ai_worker`：试卷主观题 AI 评分。
3. `reading_push_worker`：魔学院读物推送编排。
4. `deadline_reminder_worker`：试卷 / 通关临近截止提醒。

每个 worker 配套 `_stop_event`，应用 shutdown 时统一退出，并关闭共享 httpx 客户端。员工同步由管理端按需触发（HR 系统返回的数据已带 `wecom_userid`），不再保留每日企微通讯录同步任务。

## 三、后续可推进方向

### 短期

- **推送策略**：失败重试退避、限流、单用户单事件去重。
- **报表**：跨业务的学习进度 / 考试通过率 / 推送成功率看板。
- **试卷防作弊**：禁止切屏 / 强制全屏 / 时间戳水印。
- **通关历史画像**：长期能力曲线、薄弱点聚类，反哺训练推荐。

### 中期

- **多系统统一 SSO**：参见 [07_企业微信接入开发文档.md](07_企业微信接入开发文档.md) 第 10 节。
- **训练 / 通关 mode 抽象**：把复用的 chat_pipeline 进一步抽象，支持自定义剧本与判分模板。
- **知识库版本管理**：让规则更新有 review / diff / 灰度。

### 长期

- **多租户**：独立组织、独立企微 corp_id、独立知识库。
- **手机端原生体验**：小程序或 RN 包装；当前 PC + 企微 H5 体验已可用。
- **AI 学习路径推荐**：基于训练 / 通关 / 魔学院进度生成个性化下一步。

## 四、上线检查清单（每次发布参考）

- [ ] `backend/.env` 与企微回调配置匹配生产域名。
- [ ] `MAXKB_KB_ID` / API Key 在生产环境正确。
- [ ] `super_admin_*` 仅用于初始化，上线后改密。
- [ ] `WECOM_LOGIN_ENABLED` / `WECOM_PUSH_ENABLED` / `WECOM_SYNC_ENABLED` 与运营策略一致。
- [ ] `frontend/dist` 已构建并被 FastAPI 静态托管识别（或 `FRONTEND_DIST_PATH` 指向正确目录）。
- [ ] 数据库迁移脚本（`backend/sql/*.sql`）已执行。
- [ ] 新增 worker / 新事件类型在推送监控中可见。
- [ ] 验收：账号密码登录、企微登录、试卷派发推送、用户同步预览、推送监控批量删除。
