# 第一版开发排期

## 第 1 天：MaxKB 知识库准备

目标：
- 创建知识库；
- 导入所有 Markdown；
- 测试 RULE_ID 检索；
- 测试销售知识召回。

测试问题：
- 返回 RULE_ID: RULE_SCORING_RUBRIC 的完整内容
- 返回 RULE_ID: PROMPT_FINAL_REVIEW 的 PROMPT_TEMPLATE
- 初购转化训练需要考察什么？
- 客户说太贵了怎么处理？

## 第 2 天：FastAPI 基础工程

目标：
- 创建项目；
- 配置 CORS；
- 实现 `/api/health`；
- 实现 RuleLoader；
- 实现规则缓存；
- 实现 `/api/rules/reload`。

## 第 3 天：训练开始接口

目标：
- 实现 `/api/training/start`；
- 调 MaxKB 取规则；
- 调 LLM 生成训练包；
- 保存 session；
- 返回 visible_brief 和 first_customer_message。

## 第 4-5 天：聊天接口

目标：
- 实现 `/api/training/chat`；
- 每轮评分；
- 情绪更新；
- 状态机更新；
- 生成客户回复；
- 跑通至少 10 轮。

## 第 6 天：复盘接口

目标：
- 实现 `/api/training/finish`；
- 调最终复盘 Prompt；
- JSON 校验；
- 成交/未成交硬校验；
- 返回复盘结果。

## 第 7-8 天：React 前端

目标：
- 训练准备页；
- 聊天页；
- 复盘页；
- API 对接；
- 基础样式。

## 第 9 天：联调测试

测试场景：
- 初购成交；
- 初购未成交；
- 复购成交；
- 未满 10 轮结束；
- 合规风险；
- 客户强价格异议；
- 新人不探需直接报价。

## 第 10 天：优化

- Prompt 细调；
- 评分细调；
- 情绪变化细调；
- 前端体验优化。
