# React 前端页面设计

> 路由配置以 [frontend/src/App.jsx](frontend/src/App.jsx) 为准。本文档列出当前所有上线页面、对应文件与关键交互。

## 一、入口与鉴权

- `/login` → [LoginPage](frontend/src/components/LoginPage.jsx)：账号密码登录；当 `/api/auth/providers` 返回 `wecom_auto_redirect_in_client = true` 时自动跳转企微 OAuth。
- `/auth/wecom/callback` → [WecomCallbackPage](frontend/src/components/WecomCallbackPage.jsx)：处理企微回调返回的 `token` / `expires_at` / `user_json`，写入 localStorage 后跳转到原目标页。
- `/` → 已登录跳 `/admin`（管理员）或 `/home`（普通用户），未登录跳 `/login`。
- 路由守卫：`RequireAuth` / `RequireAdmin` / `RequireSuperAdmin`，统一在 `frontend/src/lib/auth.js` 维护登录态。
- 401 全局拦截：`http.js` 抛 `AUTH_REQUIRED` 后由 `App.jsx` 注册的 handler 跳 `/login`。

## 四、关键交互细节

- **训练页**：未满 `MIN_ROUNDS` 时点击结束会弹「未满 N 轮将默认未成交，是否继续？」；发送中禁用输入框；客户回复为空或失败时提示重试。
- **复盘页**：支持复制文本、跳回历史列表。
- **批量操作**：用户管理、题库、试卷、派发、通关、推送监控均支持 Table `rowSelection` + 底部/筛选栏批量删除按钮，点击带二次确认。
- **企微相关**：
  - LoginPage 在企微环境内可自动免登；登录失败时显示后端 `WecomAuthError.detail`。
  - 用户管理 Tab 提供"员工同步预览 → 执行"与"第三方员工查询"两个入口。
  - 派发面板可看到每条派发的 `wecom_push_status` 与失败原因，并可重推。
- **错误处理**：所有 `frontend/src/lib/api.*.js` 通过 `http.js` 把状态码 + detail 转为中文，组件直接 `message.error(err.message)` 即可。
- **样式**：基于 Ant Design 5 + 自定义 Tokens，深浅色主题在 `index.css` 与 `frontend/src/styles/` 控制。

## 五、隐藏字段

前端永远不展示：

- 隐藏客户画像、客户预算、底牌；
- 评分细则与成交阈值；
- 完整异议池内容；
- 推送 payload / response 的密钥级字段（仅在管理端推送详情中以原文展示，不向用户端泄露）。
