# React 前端页面设计

## 一、页面 1：训练准备页

路径：`/`

功能：
- 选择训练类型；
- 选择难度；
- 选择客户类型；
- 点击开始训练；
- 展示生成中的 loading。

字段：
- training_type：初购转化 / 复购转化 / 全链路成交
- difficulty：简单 / 中等 / 困难
- customer_type：随机 / 送礼客户 / 商务接待客户 / 自饮客户 / 企业客户 / 价格敏感客户

调用：
- POST `/api/training/start`

成功后跳转：
- `/chat/:session_id`

## 二、页面 2：模拟聊天页

路径：`/chat/:session_id`

布局：
- 左侧：训练说明与考察范围；
- 中间：聊天窗口；
- 右侧：轮次、阶段、客户情绪；
- 底部：输入框和发送按钮。

展示字段：
- 当前轮次；
- 至少 10 轮；
- 当前阶段；
- 客户状态；
- 是否可结束。

不展示：
- 隐藏客户画像；
- 客户预算；
- 评分细则；
- 成交阈值；
- 异议池完整内容。

按钮：
- 发送；
- 结束训练；
- 重新开始。

## 三、页面 3：复盘页

路径：`/review/:session_id`

展示：
- 最终结果：成交 / 未成交；
- 总分；
- 是否合格；
- 维度得分；
- 客户痛点；
- 做得好的地方；
- 不足；
- 关键转折点；
- 成交原因或未成交原因；
- 合规风险；
- 下次训练建议；
- 优化回复示例。

## 四、交互细节

- 未满 10 轮时点击结束，弹窗提示“未满 10 轮将默认未成交，是否继续？”
- 发送中禁用输入框；
- 客户回复为空或接口失败时提示重试；
- 复盘页支持复制文本。
