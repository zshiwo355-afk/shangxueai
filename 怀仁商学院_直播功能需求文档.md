# 怀仁商学院 - 公开直播/录播活动功能需求文档

> 版本：V1.1
> 日期：2026-06-15
> 核心调整：用户端公开免登录；分享出去必须以卡片形式呈现；当前以录播活动为主，同时预留真实直播能力；视频、封面、分享图均支持现上传或从素材库选择。

---

## 一、产品定位

怀仁商学院需要一个轻量级的公开直播/录播活动模块。管理员在后台快速创建活动，外部用户通过企业微信、微信或普通浏览器打开分享卡片即可观看，不进入现有登录体系。

该功能名称仍可叫“直播管理”，但产品本质是“公开播放活动”：

- 大多数活动使用录播视频。
- 少数活动后续可接入真实直播流。
- 分享链路优先，外部人点开卡片即可看。
- 后台复用现有 OSS 上传和素材库能力。

## 二、业务目标

- 管理员 3 分钟内完成一场公开活动的创建和发布。
- 外部访客点击分享卡片直接进入观看页，无需登录、无需扫码。
- 视频、封面图、分享图既可以本地上传，也可以从素材库选择。
- 分享到企业微信/微信时呈现标准卡片：标题、描述、封面图、链接。
- 先跑通录播观看和分享闭环，再扩展评论、弹幕、在线人数和真实直播。

## 三、角色与权限

| 角色 | 权限/行为 |
|------|-----------|
| 管理员/讲师 | 创建、编辑、发布、下架、删除活动；上传或选择素材；分享卡片；查看数据 |
| 外部访客 | 免登录观看已发布活动；点赞；分享；可选评论 |

关键约束：

- 公开观看页不走 `RequireAuth`。
- 公开 API 不依赖登录 token。
- 管理后台仍然必须登录并具备管理员权限。
- 企业微信 SSO 只作为后续“识别访问者”的增强能力，不作为观看门槛。

## 四、状态设计

| 状态 | 含义 | 公开页表现 |
|------|------|------------|
| `draft` | 草稿，未发布 | 不可访问 |
| `scheduled` | 已发布但未到开始时间 | 展示封面、简介、倒计时 |
| `live` | 进行中，适合真实直播或定时活动 | 展示播放器和互动 |
| `replay` | 录播/回放可看，当前最常用 | 展示播放器和互动 |
| `ended` | 已结束 | 展示结束提示，可按配置保留回放 |
| `disabled` | 已下架 | 不可访问 |

发布规则：

- 录播活动发布后默认进入 `replay`。
- 设置了未来开始时间且希望预告时，可进入 `scheduled`。
- 真实直播到开始时间后进入 `live`，结束后进入 `ended` 或 `replay`。

## 五、后台功能

### 5.1 直播列表

路由：`/admin/live`

列表字段：

- 封面缩略图
- 活动标题
- 内容类型：录播 / 直播流
- 开始时间
- 状态
- 浏览数、点赞数、分享数
- 操作：编辑、分享卡片、发布/下架、删除

筛选：

- 全部
- 草稿
- 未开始
- 进行中
- 回放中
- 已结束
- 已下架

### 5.2 新建/编辑活动

路由：

- `/admin/live/create`
- `/admin/live/:id/edit`

表单分组：

1. 基本信息
   - 活动标题，必填
   - 主讲人，选填
   - 开始时间，选填或默认当前
   - 预计时长，选填
   - 活动简介，选填

2. 视频内容
   - 内容类型：录播 / 直播流
   - 录播视频来源：现上传 / 从素材库选择
   - 直播流地址：后续真实直播使用
   - 视频上传走现有 OSS 分片上传能力
   - 素材库选择筛选 `asset_type=video`

3. 封面与详情
   - 封面图片：现上传 / 从素材库选择
   - 图片素材筛选 `asset_type=image`
   - 活动详情：轻量富文本或多行文本，首期可先支持文本/HTML

4. 分享卡片
   - 分享标题，默认活动标题
   - 分享描述，默认活动简介
   - 分享图片，默认封面图
   - 右侧实时预览卡片

5. 互动设置
   - 允许点赞，默认开启
   - 允许评论，首期可默认关闭
   - 显示浏览数/点赞数，默认开启

主要按钮：

- 保存草稿
- 发布
- 下架
- 分享卡片

## 六、公开观看页

路由：`/live/:slug`

页面要求：

- 不需要登录。
- 手机端优先，适配企业微信和微信内置浏览器。
- 第一屏清楚展示封面/视频、标题、主讲人、时间、简介。
- 主要按钮：播放、点赞、分享。
- 分享按钮优先调用企业微信/微信 JS-SDK；不支持时提供分享卡片链接。
- 已下架或不存在时显示友好错误页。

页面状态：

- 草稿/下架：不可访问。
- 未开始：显示倒计时和活动介绍。
- 录播/回放：直接可播放。
- 已结束：如保留回放则播放回放，否则显示结束。

## 七、分享卡片

卡片数据：

```json
{
  "title": "活动标题 - 怀仁商学院",
  "description": "活动简介摘要",
  "image": "https://oss.example.com/cover.jpg",
  "url": "https://example.com/live/abc123"
}
```

分享渠道：

| 渠道 | 实现方式 | 说明 |
|------|----------|------|
| 企业微信 | JS-SDK 配置分享卡片数据；分享按钮可调用 `sendChatMessage` 打开卡片分享面板 | 用户在企微内自行选择会话，发送时呈现卡片 |
| 微信 | JS-SDK 配置分享卡片数据 | 用户右上角分享时呈现卡片 |
| 普通浏览器 | Clipboard API | 提供分享卡片链接 |

实现注意：

- 直播分享不是后台主动给某个企业微信用户发消息，不需要管理员绑定企微 `userid`，也不依赖 `WECOM_ADMIN_USERIDS`。
- 后台“分享卡片”提供卡片专用 H5 地址；卡片标题、描述、图片由公开页 meta/OG 与 JS-SDK 配置共同决定。
- 公开页需要动态输出标题、描述、封面图等 meta/OG 信息，保证聊天软件抓取成卡片。
- 如果前端 SPA 无法单独满足爬虫抓取，后端 `/live/:slug` 需要返回注入 meta 的 HTML，再由 React 接管页面。

## 十、开发优先级

P0：

- 数据表和模型
- 后台 CRUD
- 视频/封面/分享图现上传
- 从素材库选择视频和图片
- 公开免登录播放页
- 分享卡片

P1：

- 分享卡片 meta/OG
- 企业微信/微信 JS-SDK 分享
- 浏览、点赞、分享统计

P2：

- 评论
- 弹幕
- 在线人数
- 真实直播流状态自动流转

## 十一、验收标准

- 管理员可以新建一个录播活动，视频和图片可从本地上传。
- 管理员可以从素材库选择视频、封面图、分享图。
- 发布后获得一个 `/live/:slug` 公开链接。
- 未登录外部用户打开链接可以观看。
- 分享出去的链接具备卡片标题、描述和图片。
- 点赞、浏览、分享数据可累计并在后台列表查看。
