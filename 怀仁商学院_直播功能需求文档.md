## 一、产品概述

### 1.1 背景

怀仁商学院需要一个轻量级直播功能，支持管理员快速创建直播、学员在企微/微信内直接观看并互动。参考小鹅通直播业务流程，但去除付费、审核限制等复杂逻辑，保留核心直播体验。

### 1.2 目标

- 管理员 3 分钟内完成一场直播的创建和发布
- 学员在企微/微信内一键进入直播间，无需扫码、无需登录
- 直播卡片支持互动（点赞、分享），增强学员参与感

### 1.3 角色

| 角色 | 说明 |
|------|------|
| **管理员/讲师** | 创建直播、管理直播、查看数据 |
| **学员** | 观看直播、互动（点赞、评论、分享） |

## 三、后台 — 直播管理

### 3.1 直播列表页

**路由：** `/admin/live/list`

| 功能 | 说明 |
|------|------|
| 新建直播 | 顶部按钮，跳转创建页 |
| 列表展示 | 表格：封面缩略图、直播名称、开播时间、状态、观看人数、操作 |
| 状态筛选项 | 全部 / 未开始 / 进行中 / 已结束 |
| 搜索 | 按直播名称搜索 |
| 操作按钮 | 编辑、复制分享链接、查看数据、删除 |

**状态说明：**

| 状态 | 触发条件 | 显示样式 |

| 已结束 | 管理员手动结束 / 到达结束时间 | 灰色标签 |

#### 3.2.2 字段定义

| 序号 | 字段名 | 类型 | 必填 | 说明 |
|:--:|--------|------|:--:|------|
| 1 | 直播名称 | VARCHAR(100) | ✅ | 显示在列表、卡片、直播间标题 |
| 2 | 开播时间 | DATETIME | ✅ | 精确到分钟 |
| 3 | 预计时长 | INT(分钟) | ❌ | 不填则不自动结束 |
| 4 | 主讲人 | VARCHAR(50) | ❌ | 讲师姓名 |
| 5 | 直播简介 | VARCHAR(200) | ❌ | 简短描述，展示在卡片 |
| 6 | 直播封面 | VARCHAR(500) | ✅ | OSS URL，16:9（900×506） |
| 7 | 直播详情 | TEXT | ❌ | 富文本 HTML |
| 8 | 分享标题 | VARCHAR(64) | ❌ | 默认取「直播名称」 |
| 9 | 分享描述 | VARCHAR(200) | ❌ | 默认取「直播简介」 |
| 10 | 分享图片 | VARCHAR(500) | ❌ | 默认取「直播封面」 |
| 11 | 开启弹幕 | BOOLEAN | — | 默认开启 |
| 12 | 允许回放 | BOOLEAN | — | 默认开启 |
| 13 | 显示在线人数 | BOOLEAN | — | 默认开启 |

#### 3.2.3 表单交互细节

1. **封面/分享图片上传**：调用阿里云 OSS 直传，上传后即时预览
2. **分享卡片预览**：右侧编辑区修改标题/描述/图片时，左侧预览卡片实时更新
3. **富文本编辑器**：轻量方案（如 Quill / TipTap），支持插入图片
4. **保存草稿**：状态为「未开始」，可继续编辑
5. **发布直播**：校验必填项 → 创建成功 → 跳转列表页 → 自动生成分享链接

#### 3.2.4 与「小鹅通」的区别

| 小鹅通有的 | 商学院不需要 | 原因 |
|-----------|:---:|------|
| 付费/免费/售卖方式 | ❌ | 商学院全部免费 |
| 商品分组关联 | ❌ | 不卖货 |
| 封面审核规则（禁二维码等） | ❌ | 无需审核，信任讲师 |
| 直播宣传封面（独立） | ❌ | 合并到分享图片 |
| 虚拟形象 | ❌ | 不搞虚的 |
| 超低延迟 | ❌ | 普通延迟够用 |
| 归属人/角色 | ❌ | 单账号操作 |
| 语音模式 | ❌ | 只做视频直播 |
| 连播设置 | ❌ | 简化 |

### 4.2 互动功能

| 功能 | 触发方式 | 反馈 |
|------|---------|------|
| **点赞** | 点击 👍 按钮 | 数字+1，飘心动画 |

| **分享** | 点击 📤 按钮 | 弹出分享面板（见 5.2） |
| **弹幕** | 评论内容 | 在视频区飘过（可开关） |
| **在线人数** | 实时 WebSocket | 右上角显示 |

### 4.3 直播状态处理

| 状态 | 页面表现 |
|------|---------|
| 未开始 | 显示倒计时 + 直播简介 + 可评论暖场 |
| 进行中 | 正常播放 + 全功能互动 |
| 已结束 | 显示「直播已结束」+ 回放入口（如有） |

## 五、分享 & 卡片设计（重点）

### 5.1 分享卡片数据结构

```json
{
  "title": "直播名称 — 怀仁商学院",
  "description": "直播简介摘要",
  "image": "https://oss.xxx.com/cover.jpg",
  "url": "https://hrxy.com/live/123?from=share"
}
```

### 5.2 分享渠道

| 渠道 | 实现方式 | 说明 |
|------|---------|------|
| **企业微信** | 企微 JS-SDK `sendAppMessage` | 直接发卡片消息到聊天窗口 |
| **微信** | 微信 JS-SDK `updateAppMessageShareData` | 微信内打开时，右上角 ... 分享 |
| **复制链接** | `navigator.clipboard.writeText()` | 兜底方案，复制纯文本链接 |

### 5.3 分享卡片在聊天中的展示效果

#### 企微聊天卡片：

```
│                              │
│  👍 128人已预约    📤 分享    │
└──────────────────────────────┘
```

#### 微信分享卡片：
```
┌──────────────────────────────┐
│   [封面小图]   直播名称        │
│               直播简介...      │
└──────────────────────────────┘
```

### 5.4 分享链接规则

- **无需扫码**：直接点击链接进入直播间
- **企微免登录**：企微环境自动走 SSO（snsapi_base 静默授权拿 UserId）
- **微信免登录**：微信环境走公众号 OAuth，首次需授权（静默），后续免感
- **外部浏览器**：需要微信内打开提示，或走手机号验证码登录

### 5.5 分享卡片互动数据

卡片上展示的互动数据：

| 数据项 | 来源 | 更新方式 |
|--------|------|---------|
| 点赞数 | 直播间实时累计 | 接口轮询 / WebSocket 推送 |
| 预约人数 | 分享链接点击 + 已预约 | 每次打开链接记录 |
| 分享数 | 分享按钮点击统计 | 分享回调上报 |

## 六、数据模型

### 6.1 数据库表设计

```sql
-- 直播主表
CREATE TABLE live_room (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    title           VARCHAR(100) NOT NULL COMMENT '直播名称',
    start_time      DATETIME NOT NULL COMMENT '开播时间',
    duration        INT DEFAULT NULL COMMENT '预计时长(分钟)',
    lecturer        VARCHAR(50) DEFAULT NULL COMMENT '主讲人',
    intro           VARCHAR(200) DEFAULT NULL COMMENT '直播简介',
    cover_url       VARCHAR(500) NOT NULL COMMENT '封面图片URL',
    detail_html     TEXT DEFAULT NULL COMMENT '直播详情(富文本)',
    share_title     VARCHAR(64) DEFAULT NULL COMMENT '分享标题',
    share_desc      VARCHAR(200) DEFAULT NULL COMMENT '分享描述',
    share_img       VARCHAR(500) DEFAULT NULL COMMENT '分享图片URL',
    enable_danmaku  TINYINT(1) DEFAULT 1 COMMENT '开启弹幕',
    enable_replay   TINYINT(1) DEFAULT 1 COMMENT '允许回放',
    show_online     TINYINT(1) DEFAULT 1 COMMENT '显示在线人数',
    status          ENUM('draft','pending','living','ended') DEFAULT 'draft',
    push_url        VARCHAR(500) DEFAULT NULL COMMENT '推流地址',
    play_url        VARCHAR(500) DEFAULT NULL COMMENT '播放地址',
    replay_url      VARCHAR(500) DEFAULT NULL COMMENT '回放地址',
    creator_id      BIGINT NOT NULL COMMENT '创建者ID',
    like_count      INT DEFAULT 0 COMMENT '点赞数',
    view_count      INT DEFAULT 0 COMMENT '观看人次',
    share_count     INT DEFAULT 0 COMMENT '分享次数',
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 直播互动记录
CREATE TABLE live_interaction (
    id          BIGINT PRIMARY KEY AUTO_INCREMENT,
    live_id     BIGINT NOT NULL,
    user_id     BIGINT NOT NULL,
    type        ENUM('like','comment','share','view') NOT NULL,
    content     TEXT DEFAULT NULL COMMENT '评论内容',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_live_user (live_id, user_id),
    INDEX idx_live_type (live_id, type)
);
```

## 八、非功能需求

### 8.1 性能
- 直播间首屏加载 < 2 秒
- 点赞/评论延迟 < 500ms
- 支持同时在线 ≥ 500 人

### 8.2 兼容性
- 企微内置浏览器 iOS/Android
- 微信内置浏览器 iOS/Android
- Chrome/Safari 最新两版

### 8.3 推流方案
推荐使用腾讯云直播（CSS）或阿里云直播，提供推流地址，讲师用 OBS 推流。
