# 训练流程规则

DOC_TYPE: rule  
VERSION: 2026-06-11  
PRIORITY: 100  
适用阶段: 全局流程  
关键词: 训练流程, start, chat, finish, 训练包, 多轮对话, 复盘, 阶段性复盘

## 一、系统流程

完整训练流程分为三段：

1. 准备训练包；
2. AI客户多轮对话；
3. 最终判断与复盘。

系统目标不是机械凑满固定轮次，而是让新人在真实销售场景中练习：
- 开场建立连接；
- 探寻客户需求；
- 建立品牌信任；
- 推荐匹配产品；
- 处理客户异议；
- 推进成交或留下下一步跟进机会；
- 遵守酒类销售合规要求。

## 二、准备训练包阶段

触发接口：`POST /api/training/start`

输入：
- training_type：初购转化 / 复购转化 / 全链路成交；
- difficulty：简单 / 中等 / 困难；
- customer_type：随机 / 送礼客户 / 商务接待客户 / 自饮客户 / 企业客户 / 价格敏感客户；
- optional_product_scope：可选产品范围。

后端应从知识库召回：
- 销售 SOP；
- 对应训练类型知识；
- 异议处理知识；
- 品牌产品白酒文化；
- 客户画像与题库规则；
- 评分规则；
- 成交判断规则；
- 合规规则；
- 准备训练包 Prompt。

准备阶段输出：
- visible_brief：展示给新人；
- hidden_training_pack：后端保存；
- first_customer_message：客户第一句话。

## 三、对话阶段

触发接口：`POST /api/training/chat`

每轮流程：
1. 记录新人输入；
2. 按每轮评分 Prompt 判断新人表现；
3. 更新 completed_actions；
4. 更新 emotion_state；
5. 更新 current_stage；
6. 根据 AI 客户对话 Prompt 生成客户回复；
7. round_count + 1；
8. 返回客户回复和可展示状态。

注意：
- 不以固定 10 轮作为成交或复盘的硬性门槛；
- 对话轮次较少时，应输出“阶段性复盘”，基于已发生内容判断；
- AI 客户只输出客户话术，不输出老师点评；
- 每轮评分结果不直接给新人看，避免泄题；
- 前端可以显示轮次、阶段和粗略客户情绪；
- 如果新人已完成关键销售动作，客户可以根据情绪与信任变化自然松动、表达意向或成交；
- 如果新人未完成关键动作，客户应继续追问、犹豫、提出异议或仅形成弱意向。

## 四、结束阶段

触发接口：`POST /api/training/finish`

结束条件：
- 用户可以在任意轮次主动结束训练；
- 系统应基于已发生的对话内容进行复盘；
- 不得因为未满固定轮次而自动判定未成交；
- 对话证据不足时，可以判定为“意向客户”或“未成交”，并说明原因；
- 出现严重合规风险时，必须判定为“未成交”。

最终结果允许：
- 成交；
- 意向客户；
- 未成交。

最终输出：
- result；
- score；
- dimension_scores；
- customer_pain_points；
- strengths；
- weaknesses；
- key_turning_points；
- lost_reason 或 deal_reason；
- next_training_focus；
- compliance_risks。

## 五、后端职责边界

后端不是业务规则维护中心，而是执行器：
- 管理 session；
- 管理轮次；
- 保存隐藏训练包；
- 调用 MaxKB 获取规则；
- 调用模型执行 Prompt；
- 校验 JSON；
- 做工程级硬校验。

业务规则、评分规则和 Prompt 模板以知识库为准。轮次只作为参考信息，不作为唯一判定条件。
