# 阶段状态机规则

RULE_ID: RULE_STAGE_MACHINE  
DOC_TYPE: rule  
VERSION: 2026-05-14  
PRIORITY: 95  
适用阶段: 对话状态控制  
关键词: 状态机, 阶段, opening, need_probe, brand_trust, product_intro, objection, closing

## 一、阶段列表

系统维护 current_stage：

1. opening：开场
2. need_probe：需求探寻
3. brand_trust：品牌信任建立
4. product_intro：产品推荐
5. price_discuss：规格价格说明
6. objection：异议处理
7. closing：成交推进
8. after_sale：发货售后
9. finished：结束

## 二、正常阶段顺序

opening
→ need_probe
→ brand_trust
→ product_intro
→ price_discuss
→ objection
→ closing
→ after_sale
→ finished

实际对话可以小幅跳转，例如客户先问价格，也可以进入 price_discuss，但销售仍应补回需求探寻。

## 三、阶段进入条件

### opening

训练开始自动进入。

目标：
- 建立连接；
- 客户提出初始问题；
- 销售回应并引导。

### need_probe

进入条件：
- 新人开始询问用途、预算、偏好、场景；
- 客户表达购买需求但不完整。

完成标志：
- asked_use_scene = true；
- asked_budget = true 或明确客户预算；
- 识别至少一个客户场景。

### brand_trust

进入条件：
- 客户质疑品牌；
- 客户问酒厂、背书、质量；
- 新人主动进行品牌价值塑造。

完成标志：
- built_brand_trust = true；
- 客户 trust 上升。

### product_intro

进入条件：
- 已有基本需求；
- 新人开始推荐产品。

完成标志：
- recommended_product = true；
- 产品推荐与客户场景匹配。

### price_discuss

进入条件：
- 客户问价格、规格、活动；
- 新人报价或解释价格。

完成标志：
- 已说明价格/规格；
- 客户价格抗拒没有持续升高。

### objection

进入条件：
- 客户提出顾虑；
- 客户犹豫、拖延、质疑。

完成标志：
- handled_objection = true；
- 至少一个核心异议得到回应。

### closing

进入条件：
- round_count >= 10；
- 已完成探需、推荐、异议处理；
- 客户出现成交信号。

完成标志：
- attempted_close = true；
- confirmed_order = true 或客户明确未成交原因。

### after_sale

进入条件：
- 客户表达成交意向或已确认订单；
- 新人说明发货、售后、收货等。

## 四、状态纠偏

如果新人跳过需求探寻直接推荐，状态可以进入 product_intro，但评分应扣需求探寻分，并让客户后续追问“你都没问我用来干嘛，怎么推荐？”

如果新人过早逼单，状态不进入 closing，客户应表现犹豫或不耐烦。
